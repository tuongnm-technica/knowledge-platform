Mục đích & Nguyên tắc thiết kế
MyGPT BA Suite được xây dựng trên một kiến trúc 4 lớp nhất quán, áp dụng xuyên suốt toàn bộ 9 agents trong pipeline. Mục tiêu cốt lõi là đảm bảo output của mỗi agent vừa machine-parseable (hệ thống có thể xử lý tự động), vừa human-readable (stakeholder và dev đọc được ngay), vừa validateable (có thể kiểm tra tính đúng đắn mà không cần chạy LLM).
Bốn lớp bao gồm: Prompt — instruction ép LLM phải output đúng cấu trúc; Machine Template — JSON skeleton mà LLM sẽ điền vào; JSON Schema — bộ quy tắc validate output trước khi đưa vào pipeline tiếp theo; và Human Template — bản markdown/table render từ JSON để người đọc tiếp nhận.
Nguyên tắc thiết kế bất biến là 4 lớp phải map 1–1 với nhau — không một field nào xuất hiện trong Human Template mà thiếu trong Schema, không một key nào LLM được phép tự thêm vào ngoài Machine Template. Điều này được enforce bởi additionalProperties: false trong toàn bộ Schema, kết hợp với việc Prompt nhúng trực tiếp skeleton JSON vào system instruction thay vì mô tả bằng ngôn ngữ tự nhiên.
Kết quả là mỗi agent hoạt động như một typed function: nhận input có cấu trúc từ agent trước, xử lý đúng phạm vi vai trò, trả ra output có cấu trúc cho agent tiếp theo — không overlap, không drift, không hallucinate field.

Cách sử dụng tài liệu này cho một hệ thống AI
Tài liệu này được thiết kế để tích hợp trực tiếp vào bất kỳ hệ thống AI nào có khả năng gọi LLM theo chuỗi — từ một chatbot đơn giản đến một orchestration framework như LangChain, AutoGen, hay custom agent loop.
Với mỗi agent (GPT-1 đến GPT-9), hệ thống AI chỉ cần thực hiện ba bước. Đầu tiên, lấy nội dung Layer 1 — Prompt và đưa vào system_message của API call. Tiếp theo, nhận output JSON từ LLM và chạy validate qua Layer 3 — JSON Schema trước khi làm bất cứ điều gì với nó — nếu validate fail, retry hoặc flag lỗi, không được để output lỗi đi tiếp pipeline. Cuối cùng, render Layer 4 — Human Template từ JSON đã validate để xuất ra giao diện người dùng hoặc lưu vào document store.
Layer 2 — Machine Template đóng vai trò là ground truth khi cần debug: nếu LLM trả về JSON thiếu field hoặc sai kiểu, đối chiếu ngay với Machine Template để xác định vấn đề nằm ở Prompt hay ở model.
Luồng dữ liệu giữa các agents rất đơn giản — intake_id từ GPT-1 trở thành intake_ref trong GPT-2, review_id từ GPT-2 trở thành review_ref trong GPT-3, và cứ thế xuyên suốt pipeline. Hệ thống AI chỉ cần truyền id của output trước vào ref field của input tiếp theo để đảm bảo traceability đầy đủ mà không cần logic phức tạp.
